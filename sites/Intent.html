Intent

16 de Noviembre de 2020

Qué es un Intent?

UN INTENT ES UN OBJETO DE ACCIÓN QUE PUEDES USAR PARA SOLICITAR UNA ACCIÓN DE OTRO COMPONENTE DE LA APLICACIÓN.

CASOS DE USO DE INTENTS


  • COMENZAR UNA ACTIVIDAD:

    Una actividad representa una única pantalla en una aplicación.

    Para iniciar una nueva instancia de una actividad pasando una INTENT a startActivity().

    La INTENT describe la actividad que se debe iniciar y contiene los datos necesarios.

    Recibir un resultado al finalizar la actividad usar starActivityForResult()


  • INICIAR SERVICIO:

    Servicio, componente que realiza operaciones en segundo plano sin una interfaz de usuario.

    Para iniciar un servicio que realice una operación única(descargar archivo) pasan una INTENT a startService().

    La INTENT describe el servicio que se debe iniciar y contiene los datos necesarios.


  • ENTREGAR UN MENSAJE:

    Mensaje, aviso que cualquier aplicación puede recibir.

    El sistema envía varios mensajes de eventos del sistema(sistema arranca o comienza carga).

    Para enviar mensajes a otras apps pasando una INTENT a sendBroadcast().

TIPOS DE INTENTS

    INTENTS EXPLÍCITAS:


  • Componentes se deben iniciar mediante su nombre(nombre de la calse completamente calificado)
  • Usuario usa una INTENT explícita para iniciar un componente en su propia aplicación ya que conoce el nombre de la actividad o servicio que desea iniciar.
  • Iniciar actividad en respuesta a la acción del usuario.

    • INTENTS IMPLÍCITAS:


  • No se nombre el componente específicamente.
  • Se declara una acción general para realizar.
  • Un componente de otr aplicación lo maneja.
  • Si desea mostrar al usuario una ubicación en un mapa, puede usar una intect para solicitar que otra aplicación capaz muestre dicha ubicación.

Acciones Locales


  • Google voice actions: “Conseguir un taxi”, “Pedir un auto”

  • Para pedir un taxi, utiliza la acción ACTION_RESERVE_TAXI_RESERVATION.

  • Nota: Las apps deben pedir confirmación del usuario antes de completar la acción.

Mapas


    Para abrir un mapa, se usa la acción ACTION_VIEW y especifica la información de ubicación en los datos de la intent con uno de los esquemas.

    Esquemas de URI datos

  • Mostrar el mapa en la longitud y latitud dadas
    • geo: latitude, longitude (Ejemplo: geo:47.6,-122.3)

  • Mostrar el mapa en la longitud y latitud dadas en un nivel de zoom determinado. (zoom= 1 muestra toda la tierra centrada en la longitud y latitud dadas, zoom=23 es el más cercano).
    • geo: latitude, longitude?z=zoom (Ejemplo: geo: 47.6,-122.3?z=11)

  • Mostrar el mapa en la longitud y latitud dadas con una etiqueta de string.
    • geo: 0,0?=lat,lng(label) (Ejemplo: geo: 0,0?q=34.99,-106.61(Treasure))

  • Mostrar la ubicación para “mi dirección postal” (puede ser una consulta de ubicación o dirección específica).
    • geo: 0,0?q=my+street+address (Ejemplo: geo:0,0?q=1600+Amphitheatre+Parkway%2C+CA)

Nota: Todas las strings deben estar codificadas, por ejemplo: 1st & Pike, Seattle. debe codificar a: 1st%20%26%20Pike%2C%20Seattle. Los espacios de ls string pueden codificarse con %20 o reeplazarse por el signo más(+).

Música o videos


  • Para reproducir un archivo de música, usa la acción ACTION_VIEW y especifica la información de ubicación en los datos de la intent.

    • Esquemas de URI datos

  • file: < URI >
  • content: < URI >
  • http: < URL >

    • Tipos de MIME(Extensiones multipropósito de correo de internet)

  • “audio/*”
  • “application/ogg”
  • “application/x-ogg”
  • “application/itunes”
  • O cualquier otro que pueda requerir la aplicación
