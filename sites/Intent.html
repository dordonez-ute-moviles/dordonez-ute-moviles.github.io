Acciones Locales


  • Google voice actions: “Conseguir un taxi”, “Pedir un auto”

  • Para pedir un taxi, utiliza la acción ACTION_RESERVE_TAXI_RESERVATION.

  • Nota: Las apps deben pedir confirmación del usuario antes de completar la acción.

Mapas


    Para abrir un mapa, se usa la acción ACTION_VIEW y especifica la información de ubicación en los datos de la intent con uno de los esquemas.

    Esquemas de URI datos

  • Mostrar el mapa en la longitud y latitud dadas
    • geo: latitude, longitude (Ejemplo: geo:47.6,-122.3)

  • Mostrar el mapa en la longitud y latitud dadas en un nivel de zoom determinado. (zoom= 1 muestra toda la tierra centrada en la longitud y latitud dadas, zoom=23 es el más cercano).
    • geo: latitude, longitude?z=zoom (Ejemplo: geo: 47.6,-122.3?z=11)

  • Mostrar el mapa en la longitud y latitud dadas con una etiqueta de string.
    • geo: 0,0?=lat,lng(label) (Ejemplo: geo: 0,0?q=34.99,-106.61(Treasure))

  • Mostrar la ubicación para “mi dirección postal” (puede ser una consulta de ubicación o dirección específica).
    • geo: 0,0?q=my+street+address (Ejemplo: geo:0,0?q=1600+Amphitheatre+Parkway%2C+CA)

Nota: Todas las strings deben estar codificadas, por ejemplo: 1st & Pike, Seattle. debe codificar a: 1st%20%26%20Pike%2C%20Seattle. Los espacios de ls string pueden codificarse con %20 o reeplazarse por el signo más(+).

Música o videos


  • Para reproducir un archivo de música, usa la acción ACTION_VIEW y especifica la información de ubicación en los datos de la intent.

    • Esquemas de URI datos

  • file: < URI >
  • content: < URI >
  • http: < URL >

    • Tipos de MIME(Extensiones multipropósito de correo de internet)

  • “audio/*”
  • “application/ogg”
  • “application/x-ogg”
  • “application/itunes”
  • O cualquier otro que pueda requerir la aplicación
